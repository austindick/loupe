# Design System Library Reference — Output Schema

This schema defines the structure of the reference file generated by the Librarian skill.
The file is designed for machine consumption by AI agents, not as human documentation.
Every section should be scannable and structured — tables, code blocks, and terse annotations.
No prose paragraphs.

## File Header

Include at the top of every generated file:

```
# [Library Name] Library Reference

> Last generated: [ISO 8601 timestamp]
> Source paths: [comma-separated list of scanned directories/files]
> Components: [count] | Tokens: [count] | Icons: [count]
```

If this is an update (not a fresh generation), also include:

```
> Last updated: [ISO 8601 timestamp]
> Changes: [count] added, [count] modified, [count] deprecated
```

## Section 1: Global Theme

Inherited CSS variables that all components share. Format as a CSS code block:

```css
/* Text */
--color-text-primary: #1A1A1A;
--color-text-secondary: #6B7280;

/* Borders */
--color-border-default: #E0E0E0;

/* Radii */
--radius-default: 4px;
--radius-small: 2px;
--radius-full: 9999px;

/* Font */
--font-family-default: "Graphik", -apple-system, sans-serif;
```

Group by category with CSS comments. Include only theme-level variables that apply globally — component-specific variables go in Section 2.

## Section 2: Components

One entry per component. **Every entry must have this exact shape:**

```markdown
## ComponentName

Import: `import { ComponentName } from '@/components/LibraryName'`
Variants: primary, secondary, ghost
Sizes: sm, md, lg
Props:
  - label: string (required)
  - variant: "primary" | "secondary" | "ghost" (default: "primary")
  - size: "sm" | "md" | "lg" (default: "md")
  - disabled: boolean (default: false)
  - icon: ReactNode (optional, renders left of label)
States: hover (built-in), disabled (opacity 0.5), active (built-in)
Gotchas: Does not accept `style` prop — use `className` or wrap in `<span>`
Usage: `<ComponentName variant="secondary" size="sm">Label</ComponentName>`
```

**Field rules:**
- **Import:** Full import statement with correct path
- **Variants:** Comma-separated list. Omit line if no variants.
- **Sizes:** Comma-separated list. Omit line if no sizes.
- **Props:** One per line, format: `name: type (required|default: value|optional[, description])`
- **States:** Comma-separated with how each manifests. Omit if no interactive states.
- **Gotchas:** Anything non-obvious from source: TODOs, HACKs, broken props, workarounds. Omit if none.
- **Usage:** Minimal JSX showing the component with its most common props

**Deprecated components** (present in reference but removed from source):

```markdown
## DeprecatedWidget [DEPRECATED]

[Original entry preserved unchanged]
```

## Section 3: Design Tokens

Grouped by category. Table format with consistent columns:

### Colors

| CSS Variable | Value | Tailwind | Usage |
|---|---|---|---|
| --color-grey-50 | #FBFBFB | bg-gray-50 | Section backgrounds |
| --color-grey-300 | #E0E0E0 | border-gray-300 | Borders, dividers |

### Spacing

| CSS Variable | Value | Tailwind | Usage |
|---|---|---|---|
| --space-sm | 8px | gap-2 | Tight grouping |
| --space-md | 16px | gap-4 | Standard gaps |

### Radii

| CSS Variable | Value | Usage |
|---|---|---|
| --radius-sm | 2px | Thumbnails, badges |
| --radius-default | 4px | Buttons, inputs, cards |

### Shadows

| CSS Variable | Value | Usage |
|---|---|---|
| --shadow-sm | 0 1px 2px rgba(0,0,0,0.05) | Cards, dropdowns |

### Z-Index

| CSS Variable | Value | Usage |
|---|---|---|
| --z-dropdown | 100 | Dropdown menus |
| --z-modal | 200 | Modal overlays |

**Column rules:**
- **CSS Variable:** The custom property name exactly as defined in source
- **Value:** Resolved value (expand references if possible)
- **Tailwind:** Closest Tailwind utility class. Omit column if project doesn't use Tailwind.
- **Usage:** Brief note on where/when to use. Inferred from variable name or usage in source.

Include a **Token Hierarchy** subsection if the system has primitive → semantic mapping:

```
### Token Hierarchy
--blue-500 (#3B82F6) → --color-interactive-primary
--grey-200 (#E5E7EB) → --color-border-default
```

## Section 5: Icons

### Built-in Icons

```
Available: ChevronDown, ChevronUp, Search, Close, Pin, Settings, ...
Import: `import { IconName } from '@/components/IconLibrary'`
Props: size ("sm" | "md" | "lg"), color (string), className (string)
```

### External Libraries

```
Library: @phosphor-icons/react
Used in codebase: ArrowRight, CaretDown, MagnifyingGlass, ...
Import: `import { IconName } from '@phosphor-icons/react'`
```

One subsection per icon library. List only icons actually used in the codebase, not the full library catalog.

## Section 6: Common Patterns

Recurring composition patterns with minimal code snippets:

```markdown
### Pattern Name

\```jsx
<ExampleCode />
\```

Notes: Brief notes on behavior, gotchas, or variations.
```

**Patterns to look for:**
- Input field styling (border, padding, focus ring)
- Dropdown/Select structure (positioning, outside-click)
- Modal composition (portal, backdrop, content)
- Form layout patterns
- Wrapper/utility components used frequently
- Card layouts and grid patterns

Only include patterns that appear 3+ times in the codebase. One code snippet per pattern.
